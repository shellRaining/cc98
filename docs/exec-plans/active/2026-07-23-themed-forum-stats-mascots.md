# 论坛统计雪球君皮肤化计划

状态：进行中

## 背景

首页右侧的论坛统计入口目前共用 `apps/website/src/assets/home/forum-stats-mascot.webp`。这张 260×260 的 WebP 来自原插图裁切，同时用于 64px 悬浮入口和 40px 面板标识。

上一轮尝试把雪球君改画成 SVG，但角色轮廓和手绘质感损失明显，方案已放弃。现有 WebP 继续作为角色标准和默认资源。后续皮肤版本只调整背景、配色和季节装饰，不重新设计雪球君。

## 目标

- 以现有 WebP 确认雪球君的基础身份，同时使用官方皮肤壁纸中的雪球君作为姿势、道具和画面风格参考。
- 为不同皮肤准备容易辨认的栅格版本，同时保证 64px 和 40px 两种显示尺寸都清楚。
- 先完成春、夏、秋、冬四个代表版本。每个版本通过人工确认后，再扩展到其余皮肤。
- 由主题资源层选择图片。`HomeForumStats.vue` 只消费当前资源，不包含具体皮肤名的判断。
- `default` 与 `summer` 共用夏季版本。专属资源缺失或加载失败时回退到现有 WebP。

## 非目标

- 不再制作或接入 SVG 版本。
- 不改变论坛统计入口的尺寸、交互、状态点和面板布局。
- 不把雪球君改成粗描边、大眼睛、圆球身体的通用兔子形象。
- 第一批不一次生成全部皮肤，也不在人工确认前把候选图接入业务代码。
- 亮暗模式默认共用同一张皮肤图，只有实际走查发现对比度问题时才增加暗色专用版本。

## 视觉约束

现有 WebP 是身份参考，官方皮肤壁纸可以作为对应皮肤的姿势、道具、场景和画面风格参考。生成时同时输入这些参考图，不使用仅靠文字描述的自由生成。

角色本体保持以下特征：

- 两只耳朵细长、接近直立，耳尖和耳根位置不变。
- 眼睛默认保持简短竖线。官方壁纸中已有明确表情时可以沿用，但不增加瞳孔、高光或睫毛。
- 允许跟随官方壁纸调整姿势和头身比例，脸型、长耳、嘴部和手绘轮廓仍需让角色一眼可认。
- 轮廓保持偏细的手绘线条，保留耳内和脸颊的排线质感。
- 角色在画面中的位置和占比接近原图，不能被道具遮住眼睛、耳朵或主要轮廓。

皮肤版本允许调整姿势、背景、主色、天气、植物和手持道具。装饰应保持少量、轮廓清楚，不放文字，不堆叠缩小后无法辨认的细节。游泳圈只属于夏季版本，其他皮肤按对应壁纸选择场景和道具。夏季优先保留现有海面和游泳圈构图，只做必要的清晰度修复。

## 验证

图片验收需要同时满足：角色特征与原图一致；64px 下能辨认雪球君和主装饰；40px 下五官和耳朵不糊；圆形裁切不截断耳朵；背景与状态点、边框和皮肤主色没有明显冲突。

浏览器走查覆盖以下场景：

- `default`、春、夏、秋、冬，以及至少一个尚无专属资源的皮肤。
- 亮色和暗色模式，包含一组亮暗配对皮肤。
- 悬浮入口的默认、悬停和聚焦状态，以及面板展开后的 40px 标识。
- 1440px 和 1140px 视口，检查破图、布局偏移和控制台错误。
- 删除或临时屏蔽一项映射，确认缺失资源会稳定回退到原图。

实施过程中每个切片完成后运行 `vp check`。全部图片和接入逻辑验收后运行 `vp run ready`。浏览器证据写入 `.artifacts/browser/2026-07-23-themed-forum-stats-mascots/`，不提交仓库。
