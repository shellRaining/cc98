# 可安装 PWA 与可靠应用外壳

## 背景

皮肤横幅缓存已经通过 PR #26 进入主线，网站目前会生成 `sw.js` 和 `registerSW.js`，并预缓存首页相关资源。第一阶段没有提供 Web App Manifest、安装图标或版本更新界面。

在最新主线重新构建后，产物共有 449 个文件，未压缩体积约 22.9 MiB。其中 JavaScript 约 3.9 MiB，CSS 约 286 KiB，图片约 16.5 MiB。全部预缓存会下载大量尚未访问的路由、编辑器、媒体播放器、代码高亮语言和图片，不适合作为安装成本。

当前预缓存清单有 15 项，共 865,201 字节。首页实际需要的 JavaScript 和 CSS 闭包有 20 项，共 783,964 字节，其中 9 项没有进入预缓存，缺少 378,750 字节。之前的离线验证保留了普通 HTTP 缓存，因此没有暴露这些缺口。清单还包含 452,278 字节的 favicon，它不影响应用启动，不应占用应用外壳预算。

## 目标

- 根据真实构建依赖图生成首页应用外壳，不再手写易漂移的 chunk 文件名。
- 清空普通 HTTP 缓存后，仅依靠 Cache Storage 仍能启动首页。
- 访问过的路由脚本、样式和字体进入有上限的运行时缓存，支持再次离线访问这些页面的代码。
- 提供 Web App Manifest 和安装图标，满足支持浏览器的安装条件。
- 页面发现新版本后显示更新提示，由用户确认后切换 Service Worker 并刷新。
- 保留现有横幅缓存，并为访问过的皮肤雪球君建立同类缓存。
- 记录应用外壳、运行时资源和业务数据的职责边界。

## 非目标

- 不预缓存全部路由、皮肤、版面图标、编辑器、播放器或代码高亮语言。
- 不缓存 CC98 API 响应，不承诺离线浏览未保存的主题和版面数据。
- 不实现离线发帖、后台同步、推送通知或安装引导弹窗。
- 不压缩正式横幅，不转换现有图片格式。
- 不承诺未访问过的路由可以在离线状态首次打开。

## 缓存边界

### 安装时预缓存

应用外壳包含 `index.html`、Service Worker 注册入口、Web App Manifest、首页入口及其静态依赖闭包、首页路由及其静态依赖闭包，以及离线首页直接显示的 Logo 和默认论坛统计雪球君。

构建配置从 Rollup 输出的入口和 `HomeView` 动态入口开始，递归收集静态 import、关联 CSS 和首页直接引用的界面图片。构建哈希变化或拆包策略调整后，清单跟随输出依赖图更新。favicon 不属于页面启动依赖，不进入应用外壳。

### 访问后缓存

其他路由的同源带哈希脚本、样式和字体使用 `NetworkFirst`，网络不可用时回退到已访问资源。生产部署会把不存在的静态文件回退成 `200 text/html`，因此不使用会永久保留错误响应的 `CacheFirst`。正常在线访问仍会先经过带哈希资源的 HTTP 长缓存。运行时缓存设置条目上限，存储配额不足时允许 Workbox 清理。

正式横幅继续使用 `skin-images-v1`。皮肤雪球君使用独立缓存，只保存访问过且响应类型正确的 WebP 图片。卡片预览图和其他普通图片不进入这两个皮肤缓存。

### 保持联网

API、头像、帖子图片、附件和第三方媒体不由本阶段 Service Worker 缓存。应用外壳离线启动后，数据请求进入现有错误状态，不显示伪造或过期的业务数据。

## 验证

生产构建需要输出应用外壳文件、字节数和依赖闭包对比。应用外壳必须覆盖入口与首页的全部静态依赖，不得包含其他 `*View` 动态入口、正式横幅、卡片预览图、整套皮肤雪球君、编辑器或播放器资源。

浏览器验收使用生产预览。首次在线访问后，清除浏览器普通 HTTP 缓存但保留 Service Worker 和 Cache Storage，再离线刷新首页。访问一个非首页路由后回到离线状态，确认其代码已进入运行时缓存。发布第二个构建版本，确认页面出现更新提示，用户确认后切换到新版本。

安装验收检查 Manifest 字段、普通图标、maskable 图标、独立窗口显示模式和启动地址。无法自动化验证的平台在结果中单独记录，不用其他浏览器结果代替。

## 风险与回滚

- 构建输出依赖图属于 Vite 和 Rollup 插件接口。实现只读取稳定的 `OutputChunk.imports`、`dynamicImports` 和 Vite 资源元数据，并用构建检查防止升级后静默漏项。
- 更新提示处理不当可能造成刷新循环。只有 `needRefresh` 为真且用户主动确认时调用更新，普通页面加载不强制刷新。
- 运行时资源缓存可能随访问增长。脚本、样式、字体和皮肤图片使用独立缓存与条目上限，配额不足时允许清理。
- 回滚时可以恢复第一阶段的注册方式和缓存规则。已经安装的旧 Worker 仍需通过后续发布完成清理和接管。

## 结果

- 构建插件会从网站入口和 `HomeView` 动态入口递归收集静态依赖、关联 CSS 和首页直接引用的图片，不再维护带哈希文件名通配清单。
- 最终构建产物有 453 个文件，约 23 MiB。预缓存清单有 24 项，共 811,162 字节，只包含首页启动闭包、Logo、默认论坛统计雪球君、`index.html` 和 Manifest。清单不含其他路由入口、favicon、安装图标、正式横幅或卡片预览图。
- 访问过的脚本、样式和字体进入 `app-assets-v1`，正式横幅和皮肤雪球君分别进入独立缓存。三类缓存都有条目上限和配额清理策略。
- Manifest 提供普通图标、maskable 图标、独立窗口显示模式和根路径启动地址。Chrome 的 Manifest 解析和安装条件检查均无错误。
- Service Worker 注册代码进入应用入口，不再生成 `registerSW.js`。新 Worker 等待时显示更新提示，用户点击“刷新”后完成接管。
- App Shell 保证首页代码和基本界面离线启动，访问过的其他路由可以复用代码。API、头像、帖子图片、附件和第三方媒体保持联网边界。

## 遗留项

- 本阶段不缓存 API 数据，离线首页会显示网络错误，不能浏览未保存的论坛内容。
- 未访问过的非首页路由不承诺首次离线打开。
- 安装入口和独立窗口外观由各平台浏览器提供，iOS、Android 和桌面系统的实际安装界面仍需发布后按平台人工抽查。
