# 冬季皮肤暗色模式试点

> 状态：已完成。冬季暗色试点已落地并通过对比度、主题切换和浏览器视觉回归。

## 背景

现有冬季皮肤沿用原版亮色横幅和主色，切到暗色模式后只继承全局深色页面。首页因此同时出现亮横幅、半透明青色顶栏、近黑页底和大面积青色卡片描边，单个颜色组合都能阅读，但整页的高对比元素过多。

当前主题 token 还让 `primary` 同时承担皮肤标识、标题文字、卡片描边、按钮底色和焦点状态。冬季主色适合做深色表面上的文字，却不适合直接承载白字，也不适合包围每一张卡片。

## 目标

- 做出一套完整可用的冬季暗色模式，保留原版横幅和冬季青色的辨识度。
- 用深灰表面层级代替近黑底色加高亮描边的组合，降低首页长时间浏览时的刺激感。
- 让顶栏、横幅、首页栏目、正文和交互状态形成稳定层级。
- 记录可复用的 token 取舍和验收结果，给后续其他皮肤改造提供依据。

## 非目标

- 本轮不调整冬季亮色模式。
- 本轮不批量改造其他皮肤，也不改变皮肤编号、日夜配对和服务端持久化逻辑。
- 不重新设计首页布局、信息密度和组件形状。
- 优先通过 CSS 处理现有横幅，不在试点阶段重绘图片资源。

## 方案

冬季暗色模式使用带少量冷蓝倾向的中性灰阶。页面、卡片和卡片内分区逐级变亮，正文使用偏灰白色，小字号正文保持充足对比度。冬季青色继续用于栏目标题、链接和焦点状态，卡片边框改为柔和的中性色，顶部标识条使用经过表面混合的冬季色。

暗色顶栏改为带冬季色调的深色表面，搜索框也使用暗色输入表面。横幅增加独立的暗色遮罩，压低雪地和灯光区域的亮度，同时保留人物与季节插画。

这轮会补充少量语义 token，至少区分普通边框与皮肤装饰边框。组件仍只读取语义 token，不判断具体皮肤名称。

## 验证

- 首页、版面页和主题页在冬季暗色模式下没有突兀的亮色大面或难以辨认的层级。
- 普通文字不低于 4.5:1，密集小字号正文以 7:1 为目标；必要的控件边界和焦点状态不低于 3:1。
- 冬季主色不再直接作为白字按钮底色，标题、链接、按钮和焦点状态均有明确可读性。
- 横幅经过暗色处理后仍能辨认原图主体，顶栏文字和搜索控件在横幅各区域都清楚可见。
- 明暗切换、皮肤切换和刷新恢复不受影响。

## 进展与调整

- 2026-07-23：基线首页确认四个主要问题：亮色横幅与深色正文跳变明显，冬季青色顶栏面积过大，首页卡片使用整圈青色描边和 8px 顶条，中性表面之间的层级较弱。
- 2026-07-23：将可读的皮肤强调色与承载白字的填充色拆分为 `primary`、`primary-fill` 和 `primary-soft`。浅色皮肤继续解析为原有值，冬季暗色单独降低填充色和装饰色的亮度与饱和度。
- 2026-07-23：冬季暗色改用冷蓝灰页面和面板层级，顶栏与搜索框改为深色表面，原版横幅增加暗色遮罩，首页栏目顶条由 8px 降为 4px。
- 2026-07-23：浏览器对照确认冬季浅色和夏季浅色没有可见回归；夏季暗色仍保持既有配色，说明试点参数没有扩散到其他皮肤。

## 验证结果

- `vp check`：格式、lint 和类型检查通过。
- 冬季暗色实测对比度：正文与面板 10.49:1，次要文字与面板 6.18:1，说明文字与面板 4.68:1，主强调文字与面板 6.65:1，白字与主填充色 4.86:1，填充色与面板 3.03:1。
- `agent-browser`：在 1440×1000 下验证冬季暗色首页、版面列表、搜索类型下拉和搜索框焦点状态；控制台无错误。
- 主题切换回归：验证冬季浅色、夏季浅色和夏季暗色，浅色主题外观保持原样，冬季暗色专属 token 不影响其他皮肤。
- 截图保存在 `.artifacts/browser/2026-07-23-dark-winter-pilot/screenshots/`，主要证据为 `final-winter-dark-home.png`、`final-winter-dark-boardlist.png`、`final-winter-dark-search-open.png` 和三张跨主题回归截图。
- 版面和主题内容接口在当前匿名会话中进入既有登录错误态，未伪造登录凭证。版面列表已覆盖密集填充色和内容面板；登录后的主题阅读页可在后续全站皮肤矩阵中抽查。
- `vp run ready`：通过。

## 结果与遗留项

冬季暗色试点达到预期：页面不再依赖近黑底和高亮青色描边制造层级，横幅仍保留原版冬季辨识度，标题、链接、控件与正文的视觉强度也有清晰区分。新增的填充色和柔和装饰色语义可以作为其他皮肤暗色适配的基础，但后续仍应逐个皮肤取色和截图验收，不宜机械套用冬季色值。

本轮没有修改界面名称、操作流程或用户权限，因此无需同步用户文档。后续推广到其他皮肤时，建议按“亮部横幅强度、表面层级、文字对比度、填充控件、焦点状态”五项建立皮肤矩阵，并补一次登录态主题阅读页抽查。

## 决策记录

- 先保留原版冬季横幅，通过遮罩验证暗色适配效果。遮罩无法兼顾主体和亮部时，再考虑单独制作暗色资源。
- 冬季青色保留为皮肤身份色，但不继续同时承担所有填充、描边和文字角色。
- 试点只形成可推广的 token 与组件规则，不提前批量修改其他皮肤。
