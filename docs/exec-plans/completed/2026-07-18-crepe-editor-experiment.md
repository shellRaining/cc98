# Crepe 编辑器体验试验

> 状态：已完成。编辑器已改用 Crepe Builder 按需组合产品层能力，保留 Markdown、上传、草稿与表单契约，并通过三个写作入口和生产构建验收。

## 背景

`MarkdownEditor.vue` 目前直接组合 Milkdown Kit 的 CommonMark、GFM、history、listener 和 upload。页面能完成发主题、回复和编辑帖子，但界面只有一排文字按钮和基础排版样式。迁移前的 `md-editor-v3` 提供图标工具栏、预览、全屏和更完整的编辑反馈，用户对比后确认当前体验退化。

前期调研测得 Crepe 完整入口约为 906.8 KB JavaScript gzip，明显大于按需组合的 Milkdown Kit。Milkdown 只在写作路由懒加载，体积增加不会进入阅读主链路，但仍需用当前 workspace 的真实构建结果判断是否可接受。

## 目标

- 使用 `@milkdown/crepe` 替换当前基础编辑器外壳，验证默认节点菜单、斜杠菜单、选区菜单和编辑画布能否改善写作体验。
- 保留 `MarkdownEditor` 的 `v-model`、禁用态、字数限制、图片上传、附件上传、错误提示和草稿恢复契约。
- 保持后端 Markdown 字符串、`contentType=1` 和 remark 阅读渲染不变。
- 使用 CC98 语义 token 调整 Crepe 主题，使编辑器适配现有皮肤与明暗模式。
- 记录写作路由构建体积，与当前 Milkdown Kit 实现和迁移前编辑器比较。

## 非目标

- 不恢复 `md-editor-v3` 或 `markdown-it`。
- 不新增协作编辑、评论、版本历史和云端草稿。
- 不为展示效果发送真实帖子、回复或编辑请求。
- 不要求保留 Crepe 的全部默认 feature。图片、数学公式和代码编辑等能力按现有 Markdown 契约取舍。

## 验收

- 编辑器默认状态不再展示整排文字按钮，常用格式可以通过选区菜单、斜杠菜单或节点菜单找到。
- 标题、段落、强调、删除线、引用、列表、任务列表、表格、代码、链接和图片能够编辑并导出 Markdown。
- 图片上传、附件插入、草稿恢复、禁用态和字数提示保持现有行为。
- 亮色、暗色和至少一个非默认皮肤下，文字、边框、菜单和焦点状态可读。
- 移动端不出现横向溢出，浮层不会被表单或页面边界裁切。
- Crepe 仍位于写作路由懒加载 chunk，阅读页面不加载编辑器依赖。
- `vp run ready` 通过，浏览器控制台没有新增错误。

## 进展与调整

- 2026-07-18：同一路由对照确认，迁移前编辑器提供约 29 个图标化操作，当前 Kit 集成只有 13 个文字按钮。当前缺口来自编辑器产品层，不是 Markdown 解析或后端契约。
- 2026-07-18：改用 `CrepeBuilder` 和 feature 子路径，启用光标、列表、链接浮层、节点菜单、选区工具栏、占位符、表格与常驻工具栏。AI 和 ImageBlock 未启用。
- 2026-07-18：ImageBlock 会把标准图片 alt 序列化为宽高比，无法保持现有 Markdown 图片语义；图片继续使用 CommonMark 节点，按钮、粘贴和拖放统一走 CC98 上传回调。
- 2026-07-18：常驻工具栏和斜杠菜单已中文化，窄屏改为单行横向滚动；主题色、暗色与非默认皮肤均从 CC98 语义 token 取值。

## 决策记录

- 2026-07-18：先用 Crepe 的真实页面效果和构建产物做判断，不因旧体积数据直接否决，也不在验证前承诺长期采用。
- 2026-07-18：首轮保留不含公式的 Crepe 按需组合方案。相较基础 Milkdown Kit，写作路由 JavaScript 增加 66.59 kB gzip，CSS 增加 14.93 kB gzip；该阶段写作 JavaScript chunk 为 173.53 kB gzip。
- 2026-07-18：用户确认需要公式能力，启用不含完整语言数据包的基础 CodeMirror 和 LaTeX feature，并在阅读侧加入 `remark-math` 与 KaTeX 渲染。ImageBlock 继续关闭。
- 2026-07-18：公式能力加入后，写作 JavaScript chunk 为 303.43 kB gzip，较不含公式的 Crepe 组合增加 129.90 kB；CSS 仍为 16.34 kB gzip。依赖继续只在写作路由按需加载。

## 验证结果

- `vp check`、website 的 281 项测试和生产构建通过；最终 `vp run ready` 通过。
- `vp pm audit -- --prod --json` 返回的 25 条既有漏洞均不经过 Milkdown 或 Crepe，新依赖链路为 0 条。
- 发主题页面验证常驻工具栏、选区工具栏、中文斜杠菜单、字数提示和草稿恢复；回复与编辑入口正常加载同一编辑器，编辑旧 Markdown 未发生图片 alt 改写。
- 只读切换后 ProseMirror 的 `contenteditable` 变为 `false`，常驻工具栏隐藏；恢复后重新可编辑。
- 亮色、暗色秋季皮肤和 390 px 窄屏显示正常，浏览器控制台没有新增错误。验收未发送真实帖子、回复或编辑请求。
- 行内公式与块公式能从 `$...$`、`$$...$$` Markdown 恢复为编辑节点，块公式使用 CodeMirror 编辑并实时预览，导出后仍保持原 Markdown 语法；斜杠菜单和常驻工具栏均提供公式入口。

## 遗留项

没有阻塞遗留。后续若增加代码语言选择器，应按语言逐项引入并测量路由体积，不直接加载 CodeMirror 完整语言数据包或 Crepe 默认 feature。
