# 架构决策记录（ADR）

记录有长期影响的架构决策：为什么选 A 不选 B，当时的环境约束是什么。

## 何时写 ADR

- 引入或替换核心依赖
- 改变分层或模块边界
- 选择数据流或状态管理范式
- 任何未来会让人（或 agent）问"为什么这样"的决定

## 何时不写

- 常规 bug 修复
- 局部实现选择（lint 会管）
- 未确定的探索（先写执行计划）

## 编号

按顺序编号：`0001-kebab-case-title.md`。一旦写入不再改号，被推翻时标 Status: Superseded 并新建一条指向后继。

## 模板

见 `template.md`。复制 `template.md` 改编号和标题开始写。
